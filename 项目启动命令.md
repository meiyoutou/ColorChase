# ColorChase 本地启动命令

> 当前项目已经强制使用 MySQL，不再支持 SQLite fallback。启动前必须保证 `.env` 已配置，MySQL 可以连接。

## 推荐启动方式

直接双击项目根目录下的：

```text
start.bat
```

或在 PowerShell 里执行：

```powershell
cd "D:\桌面\Trae临时文件\ColorChase"
.\start.bat
```

`start.bat` 会做这些事：

- 读取项目根目录的 `.env`
- 检查 `.venv312\Scripts\python.exe` 是否存在
- 检查 `COLORCHASE_DATABASE_URL` 是否已配置
- 尝试启动本机 Docker 里的 `colorchase-mysql` 容器（如果这个容器存在）
- 等待 `127.0.0.1:3306` 的 MySQL 可连接
- 启动后端：`python main.py`

## 手动启动方式

如果不用 `start.bat`，可以按下面顺序手动执行。

### 1. 确认 MySQL 可连接

```powershell
Test-NetConnection 127.0.0.1 -Port 3306
```

看到下面结果才说明 MySQL 端口通了：

```text
TcpTestSucceeded : True
```

如果你使用 Docker 里的 MySQL，可以先启动容器：

```powershell
docker start colorchase-mysql
```

如果 Docker Desktop 没启动，先打开 Docker Desktop，再执行上面的命令。

### 2. 启动后端

```powershell
cd "D:\桌面\Trae临时文件\ColorChase"
& ".\.venv312\Scripts\python.exe" main.py
```

## 访问地址

后端监听：

```text
http://127.0.0.1:8000
```

本地页面一般访问：

```text
http://127.0.0.1:8000/static/index.html
```

如果访问 `http://127.0.0.1:8000/` 出现 404 或 400，不一定代表服务没启动，先访问上面的 `/static/index.html`。

## 重启服务器

```powershell
Get-NetTCPConnection -LocalPort 8000 -ErrorAction SilentlyContinue |
  Select-Object -ExpandProperty OwningProcess |
  ForEach-Object { Stop-Process -Id $_ -Force -ErrorAction SilentlyContinue }

cd "D:\桌面\Trae临时文件\ColorChase"
.\start.bat
```

## 单行重启命令

```powershell
Get-NetTCPConnection -LocalPort 8000 -ErrorAction SilentlyContinue | Select-Object -ExpandProperty OwningProcess | ForEach-Object { Stop-Process -Id $_ -Force -ErrorAction SilentlyContinue }; cd "D:\桌面\Trae临时文件\ColorChase"; .\start.bat
```

## 停止服务器

```powershell
Get-NetTCPConnection -LocalPort 8000 -ErrorAction SilentlyContinue |
  Select-Object -ExpandProperty OwningProcess |
  ForEach-Object { Stop-Process -Id $_ -Force -ErrorAction SilentlyContinue }
```

## 常见报错

### `COLORCHASE_DATABASE_URL is missing`

说明 `.env` 里没有配置数据库地址。现在本地和生产都必须使用 MySQL。

`.env` 里需要有类似这一行：

```env
COLORCHASE_DATABASE_URL=mysql+aiomysql://colorchase:你的密码@127.0.0.1:3306/colorchase?charset=utf8mb4
```

不要把真实密码写进 README、聊天记录或 Git。

### `MySQL 127.0.0.1:3306 is not reachable`

说明 MySQL 没启动，或者端口不是 `3306`。

先检查：

```powershell
Test-NetConnection 127.0.0.1 -Port 3306
```

如果是 Docker MySQL：

```powershell
docker start colorchase-mysql
```

### `ModuleNotFoundError: No module named 'aiomysql'`

说明当前虚拟环境缺 MySQL 驱动：

```powershell
.\.venv312\Scripts\python.exe -m pip install -r requirements.txt
```

### 浏览器访问本地出现 `400 Bad Request`

如果 `.env` 里是生产模式：

```env
COLORCHASE_ENV=production
```

并且你要在本机浏览器访问 `127.0.0.1:8000`，需要让允许的 Host 包含本机地址：

```env
COLORCHASE_ALLOWED_HOSTS=colorchase.meiyoutou.top,127.0.0.1,localhost
```

否则后端会把本地浏览器请求当成不可信 Host，日志里就会出现：

```text
GET / HTTP/1.1" 400 Bad Request
```

### 浏览器打开 `/` 没有页面

优先访问：

```text
http://127.0.0.1:8000/static/index.html
```

## 备注

- 启动命令会在当前终端前台运行，占用终端。
- 停止服务可以按 `Ctrl+C`，也可以用上面的停止命令按端口结束进程。
- `start.bat` 不保存密码，密码仍然只放在 `.env` 里。
- 不要删除 `storage/`、`models/`、`weights/`、`.env`、数据库文件。
